# 单元测试系统

**完成日期**：2026-03-08

## 功能描述

为 Pomelo GPT 项目添加了完整的单元测试系统，使用 Vitest 作为测试框架，配合 Vue Test Utils 进行组件测试。所有测试都包含详细的中文注释，说明测试目的和验证内容。

## 实现细节

### 添加的依赖

在 `package.json` 中添加了以下测试相关依赖：

- `vitest`: 现代化的测试框架，与 Vite 完美集成
- `@vue/test-utils`: Vue 官方测试工具库
- `happy-dom`: 轻量级的 DOM 环境模拟
- `@vitest/ui`: Vitest 的可视化测试界面

### 测试脚本

添加了以下 npm 脚本：

```json
{
  "test": "vitest",           // 监听模式运行测试
  "test:ui": "vitest --ui",   // 使用 UI 界面运行测试
  "test:run": "vitest run"    // 单次运行所有测试
}
```

### 配置文件

创建了 `vitest.config.ts` 配置文件，包含：

- 使用 happy-dom 作为测试环境
- 配置测试文件匹配模式
- 设置覆盖率报告
- 配置路径别名

### 测试文件

创建了以下测试文件，所有测试都包含详细注释：

1. **`src/__tests__/i18n.test.ts`**
   - 测试国际化翻译功能
   - 验证中英文翻译
   - 测试嵌套键访问
   - 测试不存在键的后备处理

2. **`src/__tests__/api.test.ts`**
   - 测试 API 默认配置
   - 验证配置参数的合理性
   - 测试本地存储功能

3. **`src/__tests__/utils.test.ts`**
   - 测试字符串工具函数
   - 测试对象操作函数
   - 测试数组操作函数

### 注释规范

每个测试文件都遵循以下注释规范：

```typescript
/**
 * 测试套件描述
 * 说明这个测试套件的整体目的
 */
describe('测试套件名称', () => {
  /**
   * 单个测试用例描述
   * 详细说明这个测试验证什么功能
   */
  it('测试用例名称', () => {
    // 测试代码
  })
})
```

## 使用方法

### 安装依赖

```bash
# 使用 nvm 切换到正确的 Node 版本
source ~/.nvm/nvm.sh && nvm use

# 安装依赖
pnpm install
```

### 运行测试

```bash
# 监听模式（推荐开发时使用）
pnpm test

# 单次运行所有测试
pnpm test:run

# 使用可视化界面
pnpm test:ui
```

### 编写新测试

1. 在 `src/__tests__/` 目录下创建 `*.test.ts` 文件
2. 使用 `describe` 组织测试套件
3. 使用 `it` 编写单个测试用例
4. 为每个测试套件和用例添加详细的中文注释
5. 使用 `expect` 进行断言

### 测试文件命名规范

- 单元测试：`功能名.test.ts`
- 组件测试：`组件名.spec.ts`
- 放置位置：`src/__tests__/` 目录

## 注意事项

1. **注释要求**：每个测试套件和测试用例都必须包含详细的中文注释
2. **测试隔离**：使用 `beforeEach` 和 `afterEach` 确保测试之间相互独立
3. **覆盖率**：运行 `vitest run --coverage` 可以查看测试覆盖率
4. **异步测试**：对于异步操作，使用 `async/await` 或返回 Promise
5. **Mock 数据**：避免在测试中使用真实的 API 调用，使用 mock 数据

## 最佳实践

1. 测试应该快速、可靠、独立
2. 一个测试只验证一个功能点
3. 使用描述性的测试名称
4. 遵循 AAA 模式：Arrange（准备）、Act（执行）、Assert（断言）
5. 定期运行测试，确保代码质量
